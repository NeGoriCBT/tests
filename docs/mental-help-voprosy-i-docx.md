# Анкета Mental Help: вопросы, варианты ответов и попадание в Word

Документ описывает, какие данные собирает приложение и как они превращаются в файл **.docx**. Логика в коде: `src/mental-help-app.js` (сбор и выгрузка), `src/mental-help-data.js` (`MH_STEPS`, `MH_WORD_SECTIONS`, `buildWordBlockBody`), модули `format*ForWord` / `parse*String` в соответствующих `*-data.js` и `mental-help-v2-*.js`.

---

## 1. Как формируется итоговый Word

1. На шагах мастера ответы копятся в объекте **`answers`**: ключ шага = `step.id` (строка), значение = текст поля или JSON-строка для структурированных блоков.
2. По кнопке скачивания вызывается **`buildWordBlockBody(answers, gender, steps)`** — для каждого шага из переданного списка `steps` накапливаются фрагменты текста по трём разделам Word.
3. Порядок разделов в документе задаётся **`MH_WORD_SECTIONS`** в `mental-help-data.js` (фиксированно):

   | Порядок | Заголовок в Word (`HeadingLevel.HEADING_2`) | Ключ в `bodies` |
   |---------|---------------------------------------------|-----------------|
   | 1 | Жалобы | `complaints` |
   | 2 | Анамнез жизни | `life` |
   | 3 | Анамнез заболевания | `disease` |

4. В шапку документа попадают дата, строка для ФИО пациента, выбранный врач, пол пациента (`buildWordReportHeader` в `word-report-header.js`).
5. Внутри каждого раздела — **один или несколько абзацев** с плоским текстом: для текстовых шагов это введённый пользователем текст; для JSON-блоков — результат функций **`formatComplaintsForWord`**, **`formatLifeStructuredForWord`**, **`formatDiseaseStructuredForWord`**. Пустые разделы в Word показываются как «—».

**Важно:** данные не отправляются на сервер; docx собирается в браузере (библиотека `docx`).

## 2. Какие есть варианты анкеты (разные `steps`)

| Страница / вход | Список шагов | Особенности |
|-----------------|--------------|-------------|
| Первая версия (`mental-help.js`, по умолчанию `MH_STEPS`) | Жалобы → **текстовые** вопросы анамнеза заболевания В1–В7 → **текстовые** В1–В7 анамнеза жизни | Свободный ввод по подсказкам из `MH_STEPS`. |
| Вторая версия (`mental-help-v2.js`, `MH_STEPS_V2`) | Жалобы → те же **текстовые** В1–В7 заболевания → один шаг **`life-structured`** (структурированный анамнез жизни) | Анамнез жизни — один большой экран + JSON в `answers["life-structured"]`. |
| Только заболевание (`mental-help-disease.js`, `MH_STEPS_DISEASE_ONLY`) | Один шаг **`disease-structured`** | Структурированный анамнез заболевания, JSON в `answers["disease-structured"]`. |

Структурированный анамнез **заболевания** на основной странице v2 **не подставлен** в `MH_STEPS_V2`: там по-прежнему свободные поля В1–В7 из `MH_STEPS`.

## 3. Жалобы (`complaints`)

- **Ключ в `answers`:** `complaints` (JSON).
- **Формат хранения:** объект по id блоков из `MH_COMPLAINT_BLOCKS` (`mental-help-complaints-data.js`): у каждого блока `selected` — массив **индексов** отмеченных пунктов, `custom` — строка «свой вариант».
- **В Word:** одна строка вида **`Жалобы на: …`** — перечисление **только текстов выбранных пунктов** и непустых «свой вариант», через запятую; **без** заголовков блоков и нумерации. Дубликаты по тексту (без учёта регистра) отбрасываются. Блок 10 (менструальный цикл) показывается в форме только при поле «женский» и тогда же участвует в строке для Word.

## 4. Анамнез заболевания

### 4.1. Свободный текст (шаги `disease-v1` … `disease-v7` в `MH_STEPS`)

Текст из поля попадает в раздел Word **«Анамнез заболевания»** как отдельный абзац **без перефразирования** (если поле не пустое). Номера и формулировки вопросов в интерфейсе задаются полями `prompt` / `example` в `MH_STEPS` (`mental-help-data.js`):

| Код в интерфейсе | Смысл вопроса (кратко) |
|------------------|-------------------------|
| В1 | Когда впервые заметили симптомы; первые проявления |
| В2 | Возможные запускающие причины |
| В3 | Самостоятельные попытки справиться |
| В4 | Обращения к специалистам ранее |
| В5 | Психотропные препараты по назначению ранее |
| В6 | Динамика от начала до сейчас |
| В7 | Состояние на сегодня |

### 4.2. Структурированный шаг (`disease-structured`, страница «Анамнез заболевания»)

- **Ключ:** `disease-structured` (JSON).
- **Формирование текста:** `formatDiseaseStructuredForWord` в `mental-help-v2-disease.js` — набор **коротких связных фраз** на русском (дебют, начало, ранние симптомы, стрессоры, визиты, препараты).

**Вопрос 1 — дебют симптомов**

| Вариант | Доп. поля | Пример фразы в Word |
|---------|-----------|---------------------|
| Месяц и год | выбор месяца, года | `Дебют: <месяц> <год>.` |
| Возраст (лет) | число | `Дебют: в возрасте N лет.` |
| Не помню | — | `Дебют: не помнит / не известно.` |

**Вопрос 2 — как началось**

| Значение | Фраза в Word |
|----------|--------------|
| `gradual` | Начало постепенное. |
| `acute` | Начало острое. |
| `unknown` | Характер начала не известен. |

**Вопрос 3 — что беспокоило в начале** — свободный текст → `В начале заболевания беспокоило: …`.

**Вопрос 4 — предшествующие факторы** (мультивыбор; «Ничего из перечисленного» взаимоисключающий с остальными)

| Код | Подпись в форме | Как обобщается в тексте |
|-----|-----------------|-------------------------|
| stress | Стресс (без уточнения) | стресс |
| burnout | Психологическая перегрузка / выгорание | хроническая перегрузка |
| breakup | Расставание с партнёром / развод | расставание с партнёром |
| death | Смерть близкого человека | смерть близкого |
| jobloss | Потеря работы / увольнение | потеря работы |
| conflict | Конфликт в семье или на работе | конфликт |
| somatic | Физическая болезнь или травма | перенесённое соматическое заболевание/травма |
| pregnancy | Беременность / роды | беременность/роды |
| finance | Финансовые трудности | финансовые трудности |
| move | Смена места жительства / переезд | переезд |
| legal | Юридические проблемы / суд / тюрьма | юридические проблемы |
| none | Ничего из перечисленного | значимых предшествующих событий не было |
| other | Другое | текст из поля «Уточнение для Другое» |

Итог: `Предшествующие факторы: …` (несколько значений склеиваются через запятую и «и» по правилам `listWithAnd`).

**Вопрос 5 — обращались ли к врачу раньше**

- **Нет** → фраза с согласованием по полу: не обращался / не обращалась / не обращался(лась).
- **Да** → для каждого визита: специалист из списка + по желанию причина; в Word: «Ранее по поводу заболевания обращался(лась): … к …».

Специалисты в форме: врач психиатр; врач психотерапевт; врач невролог; психолог; терапевт / семейный врач; Другой (+ текст уточнения). Чекбокс «Не помню причину» даёт хвост «, причину не помнит».

**Вопрос 6 — назначалось ли лечение (препараты)**

- **Нет** → `Лечение по поводу заболевания ранее не назначалось.`
- **Да** → для каждого **отмеченного** препарата из каталога `DISEASE_DRUG_CATALOG` (`mental-help-disease-drugs.js`) собирается длинная фраза: молекула, торговое имя (если указано), срок в месяцах, макс. доза, эффективность (`Да` / `Частично` / `Нет` / не указано), побочные эффекты (текст или формулировка «не отмечал(а)»). Группы в каталоге: антидепрессанты; анксиолитики; нормотимики; нейролептики — полный перечень молекул и примеров ТН см. в исходном файле.

## 5. Анамнез жизни

### 5.1. Первая версия (`life-v1` … `life-v7`)

Свободный текст по вопросам из `MH_STEPS` (родственники и психиатрия; образование; работа; жизненная ситуация; тяжёлые заболевания; хроника и лекарства; алкоголь/курение/ПАВ). В Word каждый непустой ответ — отдельный абзац в разделе **«Анамнез жизни»**.

### 5.2. Вторая версия (`life-structured`)

- **Ключ:** `life-structured` (JSON).
- **Текст для Word:** `formatLifeStructuredForWord` в `mental-help-v2-life.js` — многострочный связный текст; отдельные поля превращаются в фразы с учётом **пола пациента** (где это зашито в функциях).

**Наследственность (начало блока «Рождение и семья»)**

- Три радиокнопки: **Да** / **Нет** / **Не знаю**.
- При «Нет» → фиксированная фраза «Наследственность по психическим расстройствам не отягощена.»; при «Не знаю» → «Нет объективных данных о наследственности.»; при «Да» — список случаев: для каждого задаётся **родственник** (мама, папа, брат, сестра, дедушка, бабушка, тётя, дядя, племянник, племянница), при необходимости **степень родства** (родная/двоюродная/…), **линия** (по маминой / папиной), **характер патологии** (мультивыбор + своё текстовое поле):

  - Алкогольная зависимость; Наркотическая зависимость; Депрессивные расстройства; Тревожные расстройства; Суицид (реализованный); Суицидальные попытки; Наблюдение у психиатра (формулировка согласуется с полом родственника); Деменция.

Далее на том же экране идут **блоки 1–16** (рождение и семья, раннее развитие, школа, армия только для мужского пола, образование после школы, труд, семья и дети, жильё, перенесённые заболевания из расширенного списка, операции, обмороки без ЧМТ, ЧМТ с потерей сознания, эпилепсия, хронические заболевания и постоянные лекарства, аллергии, курение/алкоголь/ПАВ). Типы элементов: да/нет, радиокнопки с фиксированными значениями, чекбоксы, числовые и текстовые поля, повторяемые списки (операции, визиты к специалистам в детстве и т.д.). Дословный перечень всех подписей и ветвлений занимает тысячи строк UI — при необходимости правки **сверяйте с `renderLifeStructuredStep`** в `mental-help-v2-life.js` (поиск по `textContent =` и `fieldset("Блок`).

## 6. Сводка ключей `answers` (для отладки и миграций)

| Ключ | Тип | Раздел Word |
|------|-----|-------------|
| `complaints` | JSON жалоб | Жалобы |
| `disease-v1` … `disease-v7` | строка | Анамнез заболевания |
| `disease-structured` | JSON | Анамнез заболевания |
| `life-v1` … `life-v7` | строка | Анамнез жизни |
| `life-structured` | JSON | Анамнез жизни |

При смене версии анкеты в одном документе могут использоваться только те ключи, которые есть в `steps` для данной страницы.
